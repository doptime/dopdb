## 0 本回合承重里程碑(一句话 + 为什么是它)

**V3:`mongostore` 在真实 MongoDB 上跑通与 `memstore` 同构的契约。** 为什么是它:整个框架到目前**只在内存 store(JSON codec)上验证过**;唯一 import 驱动的 `mongostore` 在作者沙箱里**从未编译/运行**(沙箱网络到不了驱动)。在真 Mongo + BSON 上验证之前,不能声称框架可用——这是抓 facade 的关。

## 1 范围与不在范围

**在范围**:V0 工具链+拉驱动、V1 构建(含 mongostore)、V2 无驱动测试全套、V3 mongostore 契约集成测试;外加一条文档核对并行轨。

**不在范围**:V4(httpserve 端到端在真 Mongo 后端复跑)——它依赖 V3 验过,留 R2;一切 H* 硬化项;任何对 L0 安全核心、测试、门槛的改动。

## 2 已知约束(L0/PRL、上游、🔐)

- L0 冻结见项目卡:安全语义核心(sanitize 名单、`@`-注入顺序、owner-scope 隔离、JWT 拒 none)+ 全部测试。**这些谁也不许改**。
- PRL1–PRL4 适用(`_id` 恒字符串、Find 必经净化、`@`-只来自 JWT、不放宽隔离/权限)。
- 上游:沙箱内核已自测(34 测试),但**未在真 Mongo 验过**——这是 R1 要坐实的。
- 🔐:无放宽类拍板。唯一需要人配合的是给一个隔离/一次性的测试 Mongo 连接串(`DOPTIME_TEST_MONGO_URI`);没有则 V3 走终态 B。

## 3 可预见的岔路(在 v0 里想好怎么分支,云端定稿收成决策表)

1. **mongo 驱动拉不下来**(本地网络):V0 该单元 blocked,但 V2/D1 不依赖驱动,仍要跑。
2. **`go build ./mongostore` 因 driver v2 点版本签名漂移失败**:允许**机械修正**(仅 options builder 方法名 / `mongo.Connect` 形参),禁动 `Store` 接口与 BSONCodec 行为。
3. **没有测试 Mongo / 连不上**:V3 集成测试 `t.Skip` 报因 → 终态 B,记 suspend,**不阻塞** 🟢。
4. **V3 某断言不过**(原子 Incr ≠ N、unique 未报冲突、ErrNoDoc 未命中):不改测试,记 failed + 关键数字 + 异常发现,suspend 交云端裁。

**D1 文档核对**(纯 🟢、不碰代码、不依赖驱动/ Mongo):核对 `docs/` 里的命令与路径与仓库一致,只修文档里与现实不符处。当 V3 卡在「没 Mongo」或 driver 编译时,转去干这条。
